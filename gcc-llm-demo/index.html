From stakeholder request to verified data brief — in minutes, not days.

This is the team's standard operating procedure for using Large Language Models (Gemini, Claude, ChatGPT) in routine government data-collection and analysis requests. It pairs a reusable prompting framework with a verification-first process, illustrated on two real use cases: GCC demographics and intra-GCC trade at HS-code level.

📘 Audience: Data analysts & managers
⏱ Read time: 25 min
🎯 Goal: Consistent, verifiable LLM outcomes
🔐 Data rule: No classified / PII in public LLMs
Why this matters

Compress the request-to-draft cycle

A demographic or trade brief that used to take 2–3 days of manual compilation can produce a structured first draft in 20 minutes, leaving the analyst to focus on validation, interpretation and insight — the parts a machine can't do.

What's in it

A framework, a process, two examples

The CRAFTED framework (7 elements in every prompt), a 5-phase process from intake to delivery, and two worked use cases with real prompts, real data, and the exact verification notes.

What's not in it

Magic. Blind trust. Shortcut culture.

LLMs hallucinate. In government work, an unverified number in a briefing is worse than no number at all. Every figure in this playbook carries a source, a year and a confidence marker.

How to use this document

For analysts

Read CRAFTED and Process once. Use the two use-case tabs as templates — copy the prompt, swap the scope, and follow the verification steps. Keep the prompt and raw LLM output in your project folder as the audit trail.

For managers

Jump to Use Case 1 and Use Case 2 to see the outcomes this process produces. Each tab shows the request, the prompt, the resulting data dashboard, the sources, and the verification notes. This is the quality bar the team commits to.

The workflow

The 5-phase process, from request to delivery

CRAFTED is what you write. The process is how you work. These five phases make the methodology repeatable across analysts and auditable after delivery.

PHASE 1
Frame the request
Understand the real question, the decision being supported, the granularity needed, and the data classification.
PHASE 2
Draft the prompt
In a text editor, using all 7 CRAFTED elements. Not typed directly into the chat.
PHASE 3
Execute & iterate
Run it. Refine through follow-ups. Break large tasks into chunks (one country, one chapter at a time).
PHASE 4
Verify & validate
Open cited sources. Cross-reference headline figures across two sources. Flag estimates.
PHASE 5
Package & deliver
Add methodology note. Save prompt + raw output + verified output in the project folder.

✅ Do

  • Give the LLM one filled example row when asking for a table — consistency jumps dramatically.
  • Ask for "NOT AVAILABLE" explicitly as an allowed answer.
  • Require source URL + publication year on every single row.
  • Break big asks into chunks (one country, one HS chapter).
  • Use follow-up refinement rather than re-running from scratch.

🚫 Don't

  • Ask open-ended questions like "tell me about GCC trade" — always constrain.
  • Trust suspiciously round numbers (1,000,000 / 50% / 2.5M). They are often placeholders.
  • Copy an LLM figure into a briefing without opening the source.
  • Paste classified or PII data into a public LLM — ever.
  • Skip the methodology note on the final deliverable.
Verification SLA. Briefings for senior management = 100% of figures verified. Internal working drafts = minimum 20% spot-check plus all headline figures cross-referenced. Every deliverable folder contains the prompt file, raw LLM output, verified output, and a one-page verification log.
Use case 1 · demographics

GCC demographic profile by country and variable

Stakeholder request: "We need a demographic profile of all six GCC countries — population totals, nationality split, age structure, gender, fertility, life expectancy, density, growth — for the 2026 policy briefing."

The prompt that was run

[CONTEXT] I am a data analyst in a GCC government ministry. The output will be used in a senior-management policy briefing. Accuracy and source traceability are mandatory. [ROLE] Act as a senior demographic statistician with 15+ years of experience working with GCC-Stat, UN DESA, and national statistical offices (GASTAT, FCSC, NBS, NCSI, PACI, PSA). [ACTION] Produce a comprehensive demographic profile for all six GCC countries using the most recent confirmed figures (target year: 2024, fallback 2023). Variables: total population; nationals vs non-nationals; male/female split; age bands (0–14, 15–64, 65+); urban share; median age; total fertility rate; life expectancy (M/F); annual growth %; density (per km²). [FORMAT] Return a long-format table: Country | Variable | Value | Unit | Year | Source | URL | Confidence (C/E) Plus a wide pivot for headline figures (population, nationals %, median age, TFR, life expectancy). [TARGET SOURCES] Allowed ONLY: GCC-Stat (gccstat.org), GASTAT (stats.gov.sa), FCSC (fcsc.gov.ae), iGA Bahrain (data.gov.bh), NCSI Oman (ncsi.gov.om), CSB Kuwait (csb.gov.kw), PSA Qatar (psa.gov.qa), UN DESA WPP, World Bank Open Data, GLMM factsheets. DO NOT USE Wikipedia, news blogs, or commercial databases. [EVIDENCE] Every value must carry source name, URL, and publication year. Cross-reference with a secondary source where possible; note discrepancies. [DO-NOT / GUARDRAILS] If a value is not available from the listed sources for the target year, write "NOT AVAILABLE". Do NOT estimate, interpolate or infer. Flag figures older than 2022 with (STALE). Flag estimates with (E), confirmed with (C).

Use case 2 · trade

Intra-GCC trade balance & volume — down to HS-code level

Stakeholder request: "Produce the intra-GCC bilateral trade volume and trade balance for the last three years, drilled down to HS 6-digit level for the top commodity chapters."

The prompt that was run

[CONTEXT] Government data analyst preparing intra-GCC trade analysis for a ministerial briefing. Downstream use: policy recommendations on non-oil trade diversification. [ROLE] Act as a senior trade economist specialized in GCC economic integration, with deep familiarity of UN Comtrade, WTO Stats, GCC-Stat trade bulletins and the Harmonized System (HS 2022 revision). [ACTION] Build a bilateral intra-GCC trade dataset covering all 30 country-pair directions (6 countries × 5 partners) for 2021, 2022 and 2023. Deliverables: A) Bilateral exports matrix (USD) per year B) Bilateral trade balance per year C) Top 5 HS 2-digit chapters per country-pair (by value) D) For the largest HS 4-digit heading per pair → drill-down to HS 6-digit [FORMAT] Four markdown tables A–D with explicit columns (reporter, partner, year, value USD, HS code, description, share %, source, URL, HS revision). [TARGET SOURCES] Allowed ONLY: UN Comtrade, WTO Stats, GCC-Stat Foreign Trade bulletins, ITC Trade Map, national customs/statistics authorities. Prefer reporter declaration; note mirror values. [EVIDENCE] Every value must cite source, extraction date, HS revision (2017 or 2022), and direct Comtrade query URL where applicable. [DO-NOT / GUARDRAILS] Do NOT estimate missing values — use "NOT REPORTED" and flag (MIRROR AVAILABLE) if partner data exists. Do NOT mix HS revisions without flagging. Do NOT convert currencies — report USD as given by the source.

The last gate

Quality checklist — before you hit "send"

Print this, tape it to your monitor. Nothing leaves the team's folder until every box is ticked. Click each item to mark it done.

  • Did I use all 7 CRAFTED elements in the prompt?
  • Did I specify an authoritative source whitelist?
  • Did I forbid estimation / inference and require "NOT AVAILABLE"?
  • Does every row carry a source, URL and year?
  • Did I verify at least 20% of cells against the cited source?
  • Did I cross-check headline figures against a second independent source?
  • Did I save the prompt file + raw LLM output + verified output in the project folder?
  • Did I add a methodology note disclosing that an LLM was used?
  • Is the data classification appropriate for the LLM I used (no PII / classified in public LLMs)?
  • Did I flag any figure marked (E) estimate or (STALE) for manager review?
